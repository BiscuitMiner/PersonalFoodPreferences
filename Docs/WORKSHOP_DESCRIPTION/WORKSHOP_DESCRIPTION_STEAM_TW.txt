[h1]Personal Food Preferences[/h1]

讓每位殖民者都擁有自己的料理偏好。

有些人喜歡烤肉，有些人偏愛甜點，有些人只想吃湯、海鮮、水果、炸物，甚至黑暗料理。這個模組會為 Pawn 加入個人食物偏好，當吃到偏好食物時會加心情。

如果啟用 「飲食多樣性 Dietary Variety」（預設是關閉的）則會加入挑食症機制與偏好剝奪機制，注意的是，這兩個機制是會為遊戲帶來了一些管理挑戰，所以你們可以模組設定中自行調節各項參數，如果不想要挑食病，把它關掉就好。

[h2]FAQ[/h2]

[b]Q: 此 mod 有排序要求嗎？[/b]
A: 需要放在 [b]Harmony[/b] 之後。如果你有使用 [b]RimHUD[/b] 或 [b]EdB Prepare Carefully[/b]，建議把本 mod 放在它們之後，以便相容補丁正確套用。其他食物模組通常沒有嚴格排序要求。

[b]Q: 此 mod 能否在現有存檔中途加入？[/b]
A: 可以。本模組可在現有存檔中途加入。加入後，殖民者會開始獲得料理偏好，之後進食時會依照偏好產生心情效果。建議加入後先保存一個新存檔，確認你的 modlist 沒有報錯再繼續遊玩。

移除則不建議在同一個存檔中途進行，因為本模組會在 Pawn 身上儲存偏好資料與相關狀態；中途移除可能留下缺失引用或存檔警告。

[b]Q: 是不是與 Ideology DLC 有衝突？[/b]
A: 是的，這裡指的「衝突」更多是矛盾，比如小人偏好吃肉，但戒律要求吃素，這種情況下小人就會陷入掙扎。這看上去似 Bug，但實際是可共存。

試想一下，現實中不也有那些不守清規，表裡不一的守戒人嗎？食物偏好更多指先天，小人生成時就自帶，而戒律是後天規訓，所以此 Mod 確實會為玩家帶來一些潛在挑戰。

[b]Q: 如果料理包含蔬菜、魚、肉等多食材，會如何判斷？[/b]
A: 會依照「料理本身分類」和「實際食材」一起判斷。如果一份料理同時包含蔬菜、魚、肉，則不會被視為純肉料理或全素料理，因含有魚或海鮮食材，會被標記為海鮮料理。

如果料理裡有魚，模組仍會把它作為「海鮮」相關食物來判定；如果料理本身還有其他明確分類，例如湯、燒烤、炸物、甜品等，也可能優先顯示為該料理類型。

[b]Q: 如果料理中是魚、蛋、蟲肉構成，偏好乳製品的小人會獲得好心情嗎？[/b]
A: 會。在目前判定中，蛋會被歸入乳製品分類，所以只要這道料理的食材中包含蛋，偏好「乳製品」的小人就可以透過食材標籤匹配到偏好，並獲得偏好食物的好心情。

[b]Q: 遊戲原版中，小人吃蟲肉後會感到噁心；如果小人偏好黑暗料理，那他吃包含蟲肉的料理會難受嗎？[/b]
A: 一般情況下不會。如果小人的偏好是「黑暗料理」，而食物符合黑暗料理判定，例如包含蟲肉，模組會移除原版對這類食物的部分負面心情。

所以偏好黑暗料理的小人吃含蟲肉料理時，通常不會因為原版「吃了蟲肉」而難受，並且還可以獲得偏好食物的好心情。但如果其他模組額外加入了新的負面心情，且不是原版這些 thought，則不一定會被本模組移除。

[b]Q: 能與 Vanilla Food Variety Expanded 一起使用嗎？[/b]
A: 可以的，這是兩套不同系統。VFVE 更多是直接鼓勵小人多吃不同的食物，飲食多樣化會直接加心情；PFP（本模組）則是間接的，小人吃偏好食物會開心，但也不能只吃這一類，天天吃，頓頓吃，不然會受到懲罰。

[b]Q: 如果我想修改小人的食物偏好，有什麼方法？[/b]
A: 目前有兩種方式。

[b]方法一：遊戲中開啟開發者模式修改[/b]

啟用 RimWorld 的 [b]開發者模式[/b] 後，打開小人的角色卡，點擊角色卡上顯示的「料理偏好」標籤，會開啟料理偏好分類視窗。

在一般模式下，這個視窗主要是查看目前偏好與食物分類；但在開發者模式下，可以點選其他分類，直接把該小人的偏好改成選中的類型。

簡單流程：
開啟開發者模式 → 選中小人 → 打開角色卡 → 點擊料理偏好標籤 → 在分類視窗中點選想要的新偏好

[b]方法二：使用 EdB Prepare Carefully[/b]

如果你安裝了 [b]EdB Prepare Carefully[/b]，本 mod 會在 EdB 的小人編輯介面加入一個 [b]料理偏好[/b] 選擇欄位。

你可以在開局前編輯小人時，直接從下拉選單選擇食物偏好。套用後，偏好會寫入該小人身上。

補充：目前開發者模式的 Debug Actions 裡有「檢查食物分類」、「顯示未分類食物」、「觸發偏好剝奪」等工具，但不是透過 Debug Actions 直接改偏好；實際修改入口是在角色卡的料理偏好分類視窗中。
