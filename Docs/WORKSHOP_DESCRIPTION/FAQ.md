# Personal Food Preferences FAQ

## 1. 此 mod 有排序要求嗎？

需要放在 **Harmony 之後**。

如果你有使用 **RimHUD** 或 **EdB Prepare Carefully**，建議把本 mod 放在它們之後，以便相容補丁正確套用。

簡單排序：

```text
Harmony
RimHUD                 （可選）
EdB Prepare Carefully  （可選）
Personal Food Preferences
```

其他食物模組通常沒有嚴格排序要求。

## 2. 純肉料理和全素料理是否指料理要全是肉或全是菜組成？

是的。

**純肉料理** 指料理的主要食材必須全部由肉類組成，不包含蔬菜、穀物、乳製品等其他非肉類食材。

**全素料理** 指料理不能包含肉類，也不能包含蛋、奶等畜產品；通常是由蔬菜、水果、穀物、豆類等植物性食材組成。

簡單說：

```text
純肉料理 = 全部是肉
全素料理 = 沒有肉，也沒有蛋奶等動物性產品
```

## 3. 如果料理包含蔬菜、魚、肉等多食材，會如何判斷？

會依照「料理本身分類」和「實際食材」一起判斷。

如果一份料理同時包含蔬菜、魚、肉：

```text
不會被視為純肉料理
不會被視為全素料理
如果含有魚或海鮮食材，會被標記為海鮮料理
```

原因是：

**純肉料理** 需要食材全部是肉。只要混入蔬菜、魚以外的其他非肉類食材，就不算純肉料理。

**全素料理** 不能含肉、魚、蛋、奶等動物性食材。只要含有肉或魚，就不算全素料理。

但如果料理裡有魚，模組仍會把它作為「海鮮」相關食物來判定；如果料理本身還有其他明確分類，例如湯、燒烤、炸物、甜品等，也可能優先顯示為該料理類型。

## 4. 如果料理中是魚、蛋、蟲肉構成，偏好乳製品的小人會獲得好心情嗎？

會。

在目前判定中，**蛋會被歸入乳製品 / 動物產品相關分類**。所以只要這道料理的食材中包含蛋，偏好「乳製品」的小人就可以透過食材標籤匹配到偏好，並獲得偏好食物的好心情。

但這道料理同時也會被視為含有其他特徵：

```text
魚 = 海鮮相關
蛋 = 乳製品相關
蟲肉 = 黑暗料理相關
```

所以它不會是「純乳製品料理」，但仍然可以滿足「乳製品」偏好。

## 5. 遊戲原版中，小人吃蟲肉後會感到噁心；如果小人偏好黑暗料理，那他吃包含蟲肉的料理會難受嗎？

一般情況下不會。

如果小人的偏好是 **黑暗料理**，而食物符合黑暗料理判定，例如包含蟲肉，模組會移除原版對這類食物的部分負面心情，包括：

```text
吃蟲肉
吃含蟲肉的料理
吃人肉
吃含人肉的料理
吃營養膏
吃扭曲肉
```

所以偏好黑暗料理的小人吃含蟲肉料理時，通常不會因為原版「吃了蟲肉」而難受，並且還可以獲得偏好食物的好心情。

但如果其他模組額外加入了新的負面心情，且不是原版這些 thought，則不一定會被本模組移除。

## 6. 此 mod 能否在現有存檔中途加入？

可以。

本模組可以在現有存檔中途加入。加入後，殖民者會開始獲得料理偏好，之後進食時會依照偏好產生心情效果。

建議加入後先保存一個新存檔，確認你的 modlist 沒有報錯再繼續遊玩。

移除則不建議在同一個存檔中途進行，因為本模組會在 Pawn 身上儲存偏好資料與相關狀態；中途移除可能留下缺失引用或存檔警告。

## 7. 如果我想修改小人的食物偏好，有什麼方法？

目前有兩種方式。

### 方法一：遊戲中開啟開發者模式修改

啟用 RimWorld 的 **開發者模式** 後，打開小人的角色卡，點擊角色卡上顯示的「料理偏好」標籤，會開啟料理偏好分類視窗。

在一般模式下，這個視窗主要是查看目前偏好與食物分類；但在開發者模式下，可以點選其他分類，直接把該小人的偏好改成選中的類型。

簡單流程：

```text
開啟開發者模式
選中小人
打開角色卡
點擊料理偏好標籤
在分類視窗中點選想要的新偏好
```

### 方法二：使用 EdB Prepare Carefully

如果你安裝了 **EdB Prepare Carefully**，本 mod 會在 EdB 的小人編輯介面加入一個 **料理偏好** 選擇欄位。

你可以在開局前編輯小人時，直接從下拉選單選擇食物偏好。套用後，偏好會寫入該小人身上。

補充：目前開發者模式的 Debug Actions 裡有「檢查食物分類」、「顯示未分類食物」、「觸發偏好剝奪」等工具，但不是透過 Debug Actions 直接改偏好；實際修改入口是在角色卡的料理偏好分類視窗中。

## 8. 挑食症是如何觸發的？

挑食症只會在模組設定中啟用 **飲食多樣性** 後才會生效。

啟用後，如果小人長時間只吃符合自己偏好的食物類型，模組會把這視為飲食過於單調，並累積挑食計數。達到設定門檻後，會依序觸發：

```text
輕度挑食
重度挑食
永久挑食
```

預設門檻為：

```text
連續 5 餐：輕度挑食
連續 12 餐：重度挑食
連續 20 餐：永久挑食
```

挑食會帶來身體能力與工作效率方面的負面影響；嚴重程度越高，影響越明顯。

如果小人開始吃非偏好的食物，單調飲食計數會逐步下降。輕度挑食可以恢復；重度挑食需要吃足夠次數的非偏好食物後才會降級。永久挑食一旦形成，就會保留為長期狀態。

如果你不想要挑食症、偏好剝奪或相關病症，可以在模組設定中取消啟用 **飲食多樣性**。此選項預設就是關閉的；關閉後，模組仍會保留「吃到偏好食物獲得好心情」的基本功能。
