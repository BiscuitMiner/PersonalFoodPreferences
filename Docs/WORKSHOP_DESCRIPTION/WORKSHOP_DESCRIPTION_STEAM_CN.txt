[h1]Personal Food Preferences[/h1]

让每位殖民者都拥有自己的料理偏好。

有些人喜欢烤肉，有些人偏爱甜点，有些人只想吃汤、海鲜、水果、炸物，甚至黑暗料理。这个模组会为 Pawn 加入个人食物偏好，当吃到偏好食物时会加心情。

如果启用 「饮食多样性 Dietary Variety」（预设是关闭的）则会加入挑食症机制与偏好剥夺机制，注意的是，这两个机制是会为游戏带来了一些管理挑战，所以你们可以模组设定中自行调节各项参数，如果不想要挑食病，把它关掉就好。

[h2]FAQ[/h2]

[b]Q: 此 mod 有排序要求吗？[/b]
A: 需要放在 [b]Harmony[/b] 之后。如果你有使用 [b]RimHUD[/b] 或 [b]EdB Prepare Carefully[/b]，建议把本 mod 放在它们之后，以便相容补丁正确套用。其他食物模组通常没有严格排序要求。

[b]Q: 此 mod 能否在现有存档中途加入？[/b]
A: 可以。本模组可在现有存档中途加入。加入后，殖民者会开始获得料理偏好，之后进食时会依照偏好产生心情效果。建议加入后先保存一个新存档，确认你的 modlist 没有报错再继续游玩。

移除则不建议在同一个存档中途进行，因为本模组会在 Pawn 身上储存偏好资料与相关状态；中途移除可能留下缺失引用或存档警告。

[b]Q: 是不是与 Ideology DLC 有冲突？[/b]
A: 是的，这里指的「冲突」更多是矛盾，比如小人偏好吃肉，但戒律要求吃素，这种情况下小人就会陷入挣扎。这看上去似 Bug，但实际是可共存。

试想一下，现实中不也有那些不守清规，表里不一的守戒人吗？食物偏好更多指先天，小人生成时就自带，而戒律是后天规训，所以此 Mod 确实会为玩家带来一些潜在挑战。

[b]Q: 如果料理包含蔬菜、鱼、肉等多食材，会如何判断？[/b]
A: 会依照「料理本身分类」和「实际食材」一起判断。如果一份料理同时包含蔬菜、鱼、肉，则不会被视为纯肉料理或全素料理，因含有鱼或海鲜食材，会被标记为海鲜料理。

如果料理里有鱼，模组仍会把它作为「海鲜」相关食物来判定；如果料理本身还有其他明确分类，例如汤、烧烤、炸物、甜品等，也可能优先显示为该料理类型。

[b]Q: 如果料理中是鱼、蛋、虫肉构成，偏好乳制品的小人会获得好心情吗？[/b]
A: 会。在目前判定中，蛋会被归入乳制品分类，所以只要这道料理的食材中包含蛋，偏好「乳制品」的小人就可以透过食材标签匹配到偏好，并获得偏好食物的好心情。

[b]Q: 游戏原版中，小人吃虫肉后会感到恶心；如果小人偏好黑暗料理，那他吃包含虫肉的料理会难受吗？[/b]
A: 一般情况下不会。如果小人的偏好是「黑暗料理」，而食物符合黑暗料理判定，例如包含虫肉，模组会移除原版对这类食物的部分负面心情。

所以偏好黑暗料理的小人吃含虫肉料理时，通常不会因为原版「吃了虫肉」而难受，并且还可以获得偏好食物的好心情。但如果其他模组额外加入了新的负面心情，且不是原版这些 thought，则不一定会被本模组移除。

[b]Q: 能与 Vanilla Food Variety Expanded 一起使用吗？[/b]
A: 可以的，这是两套不同系统。VFVE 更多是直接鼓励小人多吃不同的食物，饮食多样化会直接加心情；PFP（本模组）则是间接的，小人吃偏好食物会开心，但也不能只吃这一类，天天吃，顿顿吃，不然会受到惩罚。

[b]Q: 如果我想修改小人的食物偏好，有什么方法？[/b]
A: 目前有两种方式。

[b]方法一：游戏中开启开发者模式修改[/b]

启用 RimWorld 的 [b]开发者模式[/b] 后，打开小人的角色卡，点击角色卡上显示的「料理偏好」标签，会开启料理偏好分类视窗。

在一般模式下，这个视窗主要是查看目前偏好与食物分类；但在开发者模式下，可以点选其他分类，直接把该小人的偏好改成选中的类型。

简单流程：
开启开发者模式 → 选中小人 → 打开角色卡 → 点击料理偏好标签 → 在分类视窗中点选想要的新偏好

[b]方法二：使用 EdB Prepare Carefully[/b]

如果你安装了 [b]EdB Prepare Carefully[/b]，本 mod 会在 EdB 的小人编辑介面加入一个 [b]料理偏好[/b] 选择栏位。

你可以在开局前编辑小人时，直接从下拉选单选择食物偏好。套用后，偏好会写入该小人身上。

补充：目前开发者模式的 Debug Actions 里有「检查食物分类」、「显示未分类食物」、「触发偏好剥夺」等工具，但不是透过 Debug Actions 直接改偏好；实际修改入口是在角色卡的料理偏好分类视窗中。
