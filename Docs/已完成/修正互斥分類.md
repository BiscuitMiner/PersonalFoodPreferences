# PFP 代碼任務清單

## 目前任務：修正 `Meat` / `VeganMeal` 互斥分類

### 目標

修正食物分類管線中 `Meat` 與 `VeganMeal` 可同時命中的問題。

語意定義：

- `Meat`：純肉料理。主要食材全部由肉類構成；可同時具備 `Barbecue`、`Fried`、`Soup` 等烹飪形式。
- `VeganMeal`：全素料理。不含肉、魚、蛋、奶、動物產品或屍體來源食材。
- `Meat` 與 `VeganMeal` 必須互斥。
- 海鮮走 `Seafood`，不算 `Meat`。
- 蛋奶走 `Dairy` 或其他合適分類，不算 `VeganMeal`。

### 相關代碼

- `Source/FoodClassifier.cs`
  - 對外分類入口與偏好匹配。
  - 預計接入最終分類正規化。

- `Source/FoodClassificationResult.cs`
  - 保存 `PrimaryCategory`、`FallbackCategory`、`Tags`。
  - 可能需要新增安全移除 tag / 判斷互斥分類的方法。

- `Source/FoodDefAnalyzer.cs`
  - ThingDef 靜態分析。
  - 目前 keyword / override / foodType 都可能把 `Meat` 和 `VeganMeal` 同時加入 `StaticTags`。

- `Source/FoodDefAnalysis.cs`
  - 靜態分析結果。
  - 可能需要新增靜態分析層的互斥正規化。

- `Source/FoodIngredientAnalyzer.cs`
  - 實際 `Thing` 食材分析。
  - 已有 `allMeat` / `vegan` 概念，但目前只加 tag，不移除靜態誤判 tag。

- `Source/FoodPreferenceFoodListProvider.cs`
  - UI 食物列表與偏好可用性。
  - 直接讀 `StaticTags`，因此也要避免靜態 tags 出現互斥衝突。

- `Source/CompFoodPreference.cs`
  - 種族可用偏好池判斷會讀靜態分類。
  - 修復後需確認 race filter 不再因錯誤 static tag 產生錯誤可用偏好。

### SoC 方案

- 不使用 Harmony。
- 不用 XML 逐條修補全局互斥規則。
- 新增核心分類正規化邏輯，建議建立 `Source/FoodClassificationNormalizer.cs`。
- `FoodClassifier` 只負責在分類流程最後調用 normalizer。
- `FoodDefAnalyzer` 只負責在 ThingDef 靜態分析完成後調用 static normalizer。
- `FoodIngredientAnalyzer` 只負責提供實際食材判斷結果，不在內部處理 UI、心情或 Pawn 狀態。

Phase 2 設計結論：

- 新增 `Source/FoodClassificationNormalizer.cs`。
  - 類型：`internal static class FoodClassificationNormalizer`。
  - 職責：只處理分類結果一致性與 `Meat` / `VeganMeal` 互斥。
  - 禁止讀 Pawn、UI、Hediff、Settings 或單一 MOD `defName`。
  - 不承擔第三方食物語意判斷；資料錯誤仍交由 Phase 5 XML 檢查處理。

- 新增 `Source/FoodIngredientProfile.cs`。
  - 類型：`internal sealed class FoodIngredientProfile`。
  - 職責：保存一次 `CompIngredients` 掃描得到的食材摘要。
  - 建議欄位：
    - `bool HasIngredients`
    - `bool AnyMeat`
    - `bool AllMeat`
    - `bool AnySeafood`
    - `bool AnyAnimalProduct`
    - `bool AnyCorpse`
    - `bool AnyDairy`
    - `bool AnySoyProduct`
    - `bool AnyFruit`
    - `bool AnyDarkCuisine`
    - `bool IsVegan`
  - `IsVegan` 規則：有食材，且不含肉、海鮮、蛋奶 / 動物產品、屍體來源食材。
  - `AllMeat` 規則：有食材，且所有可食材都帶 `FoodTypeFlags.Meat`，不能包含海鮮、蛋奶、植物或其他非肉食材。

- 修改 `Source/FoodIngredientAnalyzer.cs` 的接口。
  - 目前：`AnalyzeInto(Thing food, FoodClassificationResult result)`。
  - 建議改為：`FoodIngredientProfile AnalyzeInto(Thing food, FoodClassificationResult result)`。
  - 若沒有 `CompIngredients` 或食材清單為空，回傳 `FoodIngredientProfile.Empty`。
  - 此方法仍負責加入食材 tags，例如 `Seafood`、`Dairy`、`SoyProduct`、`Fruit`、`DarkCuisine`。
  - 是否移除互斥 tag 不在此類處理，交給 `FoodClassificationNormalizer`。

- 修改 `Source/FoodClassificationResult.cs`。
  - 新增最小安全方法：
    - `RemoveTag(string tag)`
    - `ClearFallbackIf(string category)`
    - `ClearPrimaryIf(string category, string replacementSource = null)`
  - `ClearPrimaryIf()` 應小心使用。若移除 primary 後沒有替代分類，不應直接猜成另一個飲食類別；可讓後續 fallback 或 `GenericFood` 處理。
  - 若 primary 被清空，需同步設定 `IsUnknown = true`，並將 `PrimaryCategory` 回到 `FoodCategoryRegistry.Unknown`。

- `FoodClassificationNormalizer` 建議方法：
  - `NormalizeDefAnalysis(FoodDefAnalysis analysis)`
    - 在 `FoodDefAnalyzer.BuildDefAnalysis()` 完成 keyword / override / foodType / special case / flags 後調用。
    - 只做保守靜態修正：
      - 若 `FoodTypeFlags.AnimalProduct` 或 `FoodTypeFlags.Corpse` 存在，移除 `VeganMeal`。
      - 若同時有 `FoodTypeFlags.Meat` 與植物 / 動物產品訊號，不能保留 `Meat` + `VeganMeal` 共存。
      - 若已知 `Seafood` tag 存在，不因此添加 `Meat`。
    - 不依照 defName 猜測是否純肉或全素。

  - `NormalizeResult(FoodClassificationResult result, FoodIngredientProfile ingredients)`
    - 在 `FoodClassifier.AnalyzeFood()` 的 `FoodIngredientAnalyzer.AnalyzeInto()` 和 `ApplyFoodTypeAndGenericFallback()` 後調用。
    - 若 `ingredients.HasIngredients == false`，只做最小互斥清理，不猜測實際成分。
    - 若 `ingredients.HasIngredients == true`：
      - `ingredients.IsVegan == false` 時移除 `VeganMeal` tag / fallback / primary。
      - `ingredients.AllMeat == false` 時移除 `Meat` tag / fallback / primary。
      - `ingredients.AllMeat == true` 時允許 `Meat` tag，但仍不覆蓋 `Barbecue` / `Fried` / `Soup` 等烹飪 primary。
      - `ingredients.IsVegan == true` 時允許 `VeganMeal` tag，但仍不覆蓋 `Baked` / `Soup` / `Fried` 等烹飪 primary。

Phase 2 風險記錄：

- 靜態 `ThingDef` 無法知道可變料理實際食材，因此靜態 normalizer 必須保守。
- `FoodClassificationResult` 目前沒有移除 tag / primary / fallback 的方法；Phase 3/4 必須先加最小安全 API。
- `Meat` 與 `DarkCuisine` 的關係需要保留停機條件：人肉、昆蟲肉、扭曲肉目前優先是 `DarkCuisine`，不要在 normalizer 裡自動改成 `Meat`。
